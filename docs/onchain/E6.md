## Два контура резолва (явно)

```text
Источник истины MVP (виртуальный USD):
  admin_resolve_market → PostgreSQL status=resolved → redeem

Параллельный on-chain (testnet):
  initialize() → UMA optimistic oracle → resolve() на adapter → CTF payouts
```

Off-chain резолв **не вызывает** `resolve()` автоматически. После фиксации исхода в админке при привязанном `conditionId` статус → `pending_uma`; оператор вручную завершает on-chain (или отмечает «on-chain выполнен»).

## Схема БД

Колонки `markets.onchain_*`:

| Поле | Назначение |
|------|------------|
| `onchain_condition_id` | CTF conditionId (bytes32) |
| `onchain_question_id` | UMA questionID |
| `onchain_resolve_status` | `none` \| `pending_uma` \| `ready_onchain` \| `resolved_onchain` \| `skipped` |
| `onchain_init_tx_hash` | Tx `initialize` (опционально) |
| `onchain_resolve_tx_hash` | Tx `resolve` (опционально) |

RPC: `admin_link_market_onchain`, `admin_skip_market_onchain`, `admin_mark_onchain_resolved`.  
`admin_resolve_market` при наличии condition выставляет `pending_uma`.

## UI

- `/admin/resolve/[slug]` — блок **On-chain (UMA CTF Adapter)**: привязка condition, пропуск on-chain, отметка on-chain резолва.

## Команды

```bash
npm run onchain:smoke          # bytecode Exchange + CTF + UMA adapter
npm run onchain:uma:init       # initialize() при ONCHAIN_DEPLOYER_PK
npm test                       # tests/onchain-uma.test.ts
```

### Env (локально)

```bash
POLYGON_AMOY_RPC_URL=https://rpc-amoy.polygon.technology
ONCHAIN_DEPLOYER_PK=0x...       # funded Amoy MATIC + approve reward token
UMA_INIT_TITLE="..."
UMA_INIT_OUTCOMES=Yes,No
MARKET_SLUG=my-test-market      # подсказка в выводе скрипта
```
