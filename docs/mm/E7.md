# E7 — MM-бот / начальная ликвидность

**Статус:** off-chain MM для PostgreSQL-стакана; on-chain operator — backlog (E5/E6).

## Зачем

Пустой стакан блокирует market-ордера и ухудшает UX. MM-бот выставляет **лимитные** заявки с обеих сторон вокруг mid на открытых production-рынках (`status=open`, не sandbox).

## Архитектура

```text
npm run mm:seed  →  scripts/mm/seed-liquidity.mjs
                         │
                         ├─ service role: профиль is_mm_bot, inventory
                         └─ JWT MM user: cancel_order + place_order (RPC)
```

- Источник истины — **off-chain** matcher (`place_order`), как у всего MVP.
- Идеи стратегий — из `poly-market-maker` (OSS), **без** их инфраструктуры ([research 2026-05-20](../research/2026-05-20.md)).
- **Multi-outcome** (>2 исходов): скрипт пропускает рынок; только binary / 2-outcome.

## Котировки

| Env | Default | Смысл |
|-----|---------|--------|
| `MM_MID` | — | Фиксированный mid; иначе из стакана (чужие заявки) или 0.50 |
| `MM_SPREAD` | `0.02` | Первый уровень: bid = mid − spread, ask = mid + spread |
| `MM_STEP` | `0.02` | Шаг между уровнями |
| `MM_LEVELS` | `3` | Число уровней с каждой стороны |
| `MM_SIZE` | `25` | Размер заявки на уровень (доли) |

Логика: `scripts/mm/quote.mjs` · тесты: `tests/mm-quote.test.ts`.

## Запуск

```bash
# .env.local: SUPABASE_SERVICE_ROLE_KEY, NEXT_PUBLIC_* , MM_BOT_PASSWORD
npm run db:migrate
npm run mm:seed              # все open non-sandbox
MM_MARKET_SLUGS=btc-150k-2026 npm run mm:seed
MM_DRY_RUN=1 npm run mm:seed   # без tx
```

### Env

| Переменная | Обязательно | Описание |
|------------|-------------|----------|
| `MM_BOT_PASSWORD` | да (первый запуск) | Пароль сервисного пользователя |
| `MM_BOT_EMAIL` | нет | По умолчанию `mm-bot@forecast.local` |
| `MM_BOT_BALANCE` | нет | Баланс USD бота (`500000`) |
| `MM_INVENTORY_MIN` | нет | Мин. доли на исходе для sell (`500`) |

Пользователь создаётся через Admin API при первом запуске; `profiles.is_mm_bot = true` (миграция `20260521200000_mm_bot.sql`).

## БД

- `profiles.is_mm_bot` — exempt G2 rate limits (как `is_admin`).
- Inventory для sell: service role `upsert` в `positions` (виртуальные доли, не on-chain mint).
