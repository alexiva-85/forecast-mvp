# Админка Forecast — операторский интерфейс

> Публичный MVP: [README.md](../README.md) · Мастер-документ: [PROJECT.md](./PROJECT.md)

## Текущая версия (v1, MVP)

Админка построена вокруг **сценариев оператора**, а не «дампа БД».

| Раздел | URL | Назначение |
|--------|-----|------------|
| Обзор | `/admin` | Счётчики, оборот 24ч/7д/30д, топ рынков, очередь «требует действия» |
| Рынки | `/admin/markets` | Вкладки по статусу, поиск, карточки с метриками |
| Создать | `/admin/markets/new` | Мастер из 3 шагов + тестовый флаг |
| Редактировать | `/admin/markets/[slug]/edit` | Текст, даты, правила резолва; slug только в черновике (A2) |
| Резолв | `/admin/resolve` | Очередь `closed` → отдельный экран фиксации |
| Идеи | `/admin/ideas` | Gamma (read-only) → черновик в один клик или мастер |
| Пользователи | `/admin/users` | Модерация, KYC-заготовка, лимиты G2 (A11) |
| Сделки | `/admin/trading` | Сделки и заявки: поиск, фильтр статуса (A12) |
| Журнал | `/admin/audit` | Лог действий операторов (A4) |
| Выводы | `/admin/withdrawals` | Очередь заявок на вывод, сверка ledger (E4) |
| Жалобы | `/admin/reports` | Очередь жалоб пользователей (D6) |
| Настройки | `/admin/settings` | Комиссия, история ставки (A13), лимиты; SQL только в dev-блоке |

### Жизненный цикл рынка (сейчас)

```text
draft  →  open  →  closed  →  resolved
          ↑ «Опубликовать» (production)
          ↑ auto (closes_at) или «Закрыть торги»
```

- **Черновики**: `status = draft` — не в каталоге; публикация через «Опубликовать» (RPC `admin_publish_draft_market`).
- **Тестовые рынки**: `is_sandbox = true` — скрыты из каталога; «В каталог» снимает флаг sandbox (RPC `admin_publish_market`), статус остаётся `open`.
- **Резолв** возможен только для `status = closed` (RPC `admin_resolve_market`).
- **Метрики в админке**: RPC `admin_market_stats` (оборот за всё время, сделки, открытые ордера); `admin_platform_volume` / `admin_top_markets_by_volume` (оборот по периодам и топ рынков).
- **Напоминания резолва (A15)**: баннер на всех страницах админки и счётчик в пункте «Резолв», если есть `closed` рынки (без email — без внешних сервисов).

### Доступ

Флаг `profiles.is_admin`. Выдача прав — в Supabase SQL (см. collapsible «Для разработчиков» в Настройках).
