# UX Forecast — правила интерфейса

> Черновик-заготовка. Расширять по мере приёмки экранов.  
> Агент при UI-задачах читает этот файл **вместе с** [AGENT_WORKFLOW.md](./AGENT_WORKFLOW.md).

---

## 1. Принцип

Пользователь должен за несколько секунд понять: **что я сделал · с каким исходом · сколько · на какую сумму**.  
Не «биржевой журнал» (одна строка через точки, технический порядок полей).

Референс по смыслу: [Polymarket](https://polymarket.com) — **не** копия визуала 1:1.

---

## 2. Иерархия строк

| Уровень | Содержание | Пример |
|---------|------------|--------|
| 1 | Действие / тип | Продажа, Купить: Команда A |
| 2 | Рынок · исход · условия | Bitcoin… · Да · 10 долей по 50¢ |
| Справа | Сумма или кнопка | +$2.49 / Отменить |

В коде: компонент `UiListRow` (`src/components/UiListRow.tsx`) — единая разметка для кабинета, стакана, сделок и истории.

**Заявки:** глагол первым — «Купить» / «Продать», не «Покупка» (заявка ещё не исполнена).

**Цвет суммы:** по знаку денег (+ зелёный, − нейтральный), не по слову «Продажа».

---

## 3. Кабинет

Порядок разделов: **обзор → позиции → заявки → история** (не история внизу одной простыни).

Меню кабинета — справа на desktop; на mobile — горизонтальные вкладки сверху контента.

Маршруты: `/portfolio`, `/portfolio/deposit`, `/portfolio/withdraw`, `/portfolio/referral`, `/portfolio/positions`, `/portfolio/orders`, `/portfolio/activity`, `/profile`.

**Рефералы (F2):** `/portfolio/referral` — код и ссылка `?ref=`, бонус $500 обоим (тестовые деньги), ввод кода в течение 7 дней после регистрации.

**Пополнение / вывод (E3–E4):** дисклеймер «тестовые деньги»; пополнение — описание режима; вывод — заявка резервирует сумму на счёте (E4 ledger), список со статусом, отмена pending; оператор — `/admin/withdrawals`.

---

## 4. Исходы

- Binary: **Да** / **Нет**
- Multi: только подписи исходов из `market_outcomes`, без ложного «Нет» как проигрыша
- Склонение: 1 доля / 2 доли / 5 долей; цена: «по 50¢», не `@ 50¢` в пользовательском UI

---

## 5. Статусы рынка (RU)

Использовать `marketStatusLabel`: **Открыт** · **Торги закрыты** · **Завершён** — не `open` / capitalize EN.

---

## 6. Приёмка UI (чеклист для агента)

- [ ] Главное действие видно без чтения всей строки
- [ ] Нет обязательного скролла ради частой операции (история — отдельный маршрут)
- [ ] Проверка на multi- и binary-рынке
- [ ] Тексты на русском, без жаргона БД (side, slug — только в админке/дебаге)
